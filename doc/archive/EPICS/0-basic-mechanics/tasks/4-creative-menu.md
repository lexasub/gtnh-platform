# TASK: Креативное меню (Creative Menu)

**Layer**: 1
**Статус**: Draft
**Эпик**: 0-basic-mechanics

## Affected Services

| Service | Role | R/W |
|---------|------|-----|
| **GameClient** | Primary — ImGui окно со списком предметов | Read/Write |
| **Gateway** | Relay — forwards запрос на получение предмета | Relay |
| **SimulationCore** | Выдача предмета в инвентарь | Write |

---

## Описание

Без инструментов и рудодобычи игроку нужен способ получать предметы для тестирования крафтов и машин. Креативное меню — ImGui-окно со списком всех зарегистрированных предметов/блоков.

---

## Требования

### Интерфейс
- ImGui-окно со списком всех зарегистрированных предметов/блоков
- Поиск/фильтрация по имени (опционально на MVP)
- Выбор предмета и количества

### Функциональность
- Выбранный предмет появляется в инвентаре игрока
- Количество: по умолчанию 1, можно изменить (1–64)
- Если инвентарь полон — предмет не добавляется

### Открытие
- Клавиша открытия: TBD (например, **C** или **/**)
- Можно открывать в любой момент

### Архитектура
```
Client: открывает CreativeMenu → выбирает предмет
    → CraftRequest/PlayerAction (creative_give)
    → Gateway → SimulationCore
    → SimulationCore: добавляет предмет в инвентарь
    → InventoryUpdate → Gateway → Client
```

---

## Acceptance Criteria

#### Сценарий: Открыть креативное меню
1. Игрок нажимает клавишу **(TBD)**
2. Открывается ImGui-окно со списком предметов
3. В списке: stone, dirt, iron_ore, coal, furnace, crafting_table и т.д.

#### Сценарий: Выдать предмет
1. Игрок выбирает "iron_ingot" в креативном меню
2. Указывает количество: 10
3. Нажимает "Выдать"
4. 10 iron_ingot появляются в инвентаре
5. Меню остаётся открытым

#### Сценарий: Инвентарь полон
1. Инвентарь полностью заполнен
2. Игрок пытается выдать предмет через креатив
3. Предмет не добавляется
4. Игрок видит сообщение "Инвентарь полон"
