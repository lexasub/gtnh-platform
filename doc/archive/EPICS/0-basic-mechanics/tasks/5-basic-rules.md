# TASK: Базовые правила MVP

**Layer**: 1
**Статус**: Draft
**Эпик**: 0-basic-mechanics

---

## Описание

Набор правил и упрощений для MVP, чтобы минимизировать объём реализации и сфокусироваться на базовых механиках.

---

## Правила

| Правило | Описание | Обоснование |
|---------|----------|-------------|
| **Нет дропа** | Сломал блок → предмет сразу в инвентарь. Никаких выпавших предметов на землю. | Не нужно реализовывать физику дропа, анимацию, сбор предметов с земли, despawn таймеры |
| **Нет инструментов** | Любой предмет/рука ломает блок с одинаковой скоростью. Нет типов инструментов (кирка, топор, лопата). | Не нужна система прочности, типы инструментов, скорость копания |
| **Нет прочности** | Предметы не расходуются (кроме блоков при постройке). Нет耐久. | Не нужно реализовывать систему прочности, ремонт, зачарования |
| **Полное доверие клиенту** | Клиент сообщает действие (поставил блок, сломал, переместил предмет) — сервер не проверяет валидность, только применяет. | Античит, валидация состояний — отложены |
| **Весь GUI — ImGui** | Все окна интерфейса рисуются через ImGui. Без кастомного рендера, без HTML/JS. | ImGui — быстрейший способ прототипирования |
| **Энергия — число** | Машины работают "на магии" — бесконечная энергия. Энергия просто число, нет проводов, потерь, сетей. | Энергосети, PipeNetwork, потери — отложены (Layer 2) |
| **Крафт** | Крафт ТОЛЬКО через блок верстака. Нет крафта в инвентаре (2×2). | Упрощение — не нужно реализовывать два разных UI крафта |
| **Нет руки/тела** | Не рендерим игрока (руку, тело, модель). Только UI. | Упрощение рендеринга, не нужна модель игрока |

---

## Где применяется

| Сервис | Какие правила применяет |
|--------|------------------------|
| **SimulationCore** | Нет инструментов, нет прочности, нет дропа, полное доверие — валидация действий игрока |
| **GameClient** | Весь GUI — ImGui, нет руки/тела — рендеринг и UI |
| **ChunkStore** | Нет дропа — блок сразу в инвентарь, ChunkStore не участвует |
| **Gateway** | Полное доверие — просто релей, без валидации |
| **PipeNetwork** | Энергия — число (заглушка, ничего не делает) |

---

## Примечания

- Правила могут быть пересмотрены после MVP
- Каждое правило — осознанное упрощение для скорости разработки
- При добавлении новых механик проверять соответствие этим правилам
