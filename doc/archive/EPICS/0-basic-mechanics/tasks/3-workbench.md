## Описание

Первая машина, доступная игроку на MVP — **верстак** (CRAFTING_TABLE). Позволяет открыть GUI с сеткой 3×3, переносить предметы и нажимать кнопку крафта (которая пока ничего не делает).

---

## Требования

### Блок верстака
- Новый тип блока: `CRAFTING_TABLE` (block_id: 100)
- Блок ставится как обычный блок из инвентаря
- ПКМ по верстаку → открывается ImGui окно

### GUI верстака
- Сетка крафта **3×3** (9 слотов)
- Слот результата (справа от сетки)
- Кнопка "Craft" (присутствует, но крафт НЕ совершается)
- Результат крафта не показывается (заглушка)

### Взаимодействие с инвентарём
- Перетаскивание предметов между инвентарём игрока и сеткой верстака
- На MVP: упрощённый клик (выбрал слот → предмет переместился)

### Открытие/Закрытие
- **Открыть**: ПКМ по блоку верстака
- **Закрыть**: Escape / ПКМ по другому блоку
- При закрытии содержимое сетки НЕ теряется

### Персистентность
- Состояние сетки хранится в EntityStateStore по ключу `(dim, x, y, z)`
- При повторном открытии предметы восстанавливаются
- Чанк выгрузился → состояние в EntityStateStore сохранилось

```flatbuffers
table BlockEntityUpdate {
    x: uint32; y: uint32; z: uint32;
    machine_type: uint16;                // CRAFTING_TABLE = 100
    inventory: [ItemStack];              // все слоты

}
```

## Acceptance Criteria

#### Сценарий: Открыть верстак
1. Игрок ставит блок верстака из инвентаря
2. ПКМ по верстаку
3. Открывается ImGui окно с сеткой 3×3 + слот результата + кнопка Craft
4. Все слоты пусты

#### Сценарий: Переместить предмет в сетку
1. Открыт верстак
2. Игрок кликает по слоту с углём в инвентаре
3. Уголь перемещается в первый слот сетки верстака
4. В слоте отображается иконка угля

#### Сценарий: Кнопка Craft (stub)
1. В сетку 3×3 помещены ингредиенты
2. Игрок нажимает "Craft"
3. Ничего не происходит (или сообщение "Craft будет позже")
4. Предметы остаются в сетке

#### Сценарий: Сохранение и восстановление
1. В сетку верстака помещены предметы
2. Игрок закрывает окно (Escape)
3. Игрок снова открывает верстак (ПКМ)
4. Предметы всё ещё в сетке

#### Сценарий: Закрыть через другой блок
1. Открыт верстак
2. Игрок нажимает ПКМ по другому блоку
3. Окно верстака закрывается
4. Открывается окно другого блока (если есть)

## Done (MVP)

- **WorkbenchWindow** — 3×3 grid, result slot, "Craft" button (stub), player inventory display
- **Click-to-move** — упрощённая перетаскивание: клик по слоту перемещает предмет из инвентаря
- **BlockUIFactory** — регистрация через FindOrCreate(BlockType::CRAFTING_TABLE)
- **EntityStateStore persistence** — состояние сетки сохраняется по ключу (dim, x, y, z), восстанавливается при повторном открытии
- **Open/close** — ПКМ по верстаку → открыто, Escape/ПКМ по другому блоку → закрыто, содержимое сохраняется
- **BlockType::CRAFTING_TABLE = 100** — новый тип блока
